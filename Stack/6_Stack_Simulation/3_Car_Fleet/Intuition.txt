Question Link : https://leetcode.com/problems/car-fleet/description/

*   Problem:
        Cars move toward target

        Faster cars may catch slower ones → form fleet

*   Goal:
        Count number of fleets

*   Key idea:

        Compute time to reach target

*   Why time?

        Fleet is determined by arrival time

*   Step 1:

        time = (target - position) / speed

*   Step 2:

        Sort cars by position DESC
        (closest to target first)

*   Why?

        So we process "front → back"

*   Step 3:

        Use stack to track fleet times

*   Logic:

        If current time > top:
            forms new fleet

        If current time <= top:
            merges into fleet ahead

*   Important invariant:

        stack is MONOTONIC DECREASING

*   Time Complexity:

        O(n log n)

*   Space Complexity:

        O(n)
