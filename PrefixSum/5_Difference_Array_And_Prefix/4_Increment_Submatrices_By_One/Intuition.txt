Question Link : https://leetcode.com/problems/increment-submatrices-by-one/description/

*   Problem:
        Apply multiple rectangle increment queries on a matrix.

*   Brute force:

        For each query:
            update all cells in rectangle

        Time Complexity:
            O(q * n^2) → too slow

*   Key observation:

        2D range updates can be optimized
        using difference array.

*   Core idea:

        Extend 1D difference array to 2D.

*   2D Difference array updates:

        For rectangle (x1, y1) → (x2, y2):

            diff[x1][y1]       += val
            diff[x2+1][y1]     -= val
            diff[x1][y2+1]     -= val
            diff[x2+1][y2+1]   += val

*   Why 4 updates?

        They ensure effect applies only inside rectangle.

*   Visualization:

        + at top-left
        - at boundaries
        + at bottom-right (to fix overlap)

*   Step 2:

        Convert diff → actual matrix using 2D prefix sum.

*   2D prefix formula:

        matrix[i][j] =
            diff[i][j]
            + matrix[i-1][j]
            + matrix[i][j-1]
            - matrix[i-1][j-1]

*   Why subtract top-left?

        It gets counted twice from top and left.

*   Important invariant:

        diff stores changes,
        prefix sum reconstructs final values.

*   Time Complexity:

        O(n^2 + q)

*   Space Complexity:

        O(n^2)
