Question Link : https://leetcode.com/problems/count-the-number-of-fair-pairs/description/

*   We are given an array nums and two integers:
        lower
        upper

    We must count number of pairs (i, j)
    such that:

        i < j
        lower <= nums[i] + nums[j] <= upper

*   Direct counting in range is difficult.

*   Key transformation:

        Count pairs with sum <= upper
        minus
        Count pairs with sum <= lower - 1

        This gives count of sums in [lower, upper].

*   So problem reduces to:

        Efficiently count pairs with sum <= K.

*   Strategy for counting sum <= K:

        (1) Sort the array.

        (2) Use two pointers:
                left = 0
                right = n - 1

        (3) While left < right:

                If nums[left] + nums[right] > K:
                        right--

                Else:
                        count += (right - left)
                        left++

*   Why count += (right - left)?

        If nums[left] + nums[right] <= K,
        then for that fixed left:

            nums[left] + nums[k] <= K
            for all k in (left+1 ... right)

        Because array is sorted,
        and any smaller index gives smaller sum.

*   Important invariant:

        Sorted order ensures:
            Moving right decreases sum.
            Moving left increases sum.

        Each pointer moves at most n times.

*   Time Complexity:
        Sorting → O(n log n)
        Two pointer traversal → O(n)

        Overall → O(n log n)

*   Space Complexity:
        O(1)
