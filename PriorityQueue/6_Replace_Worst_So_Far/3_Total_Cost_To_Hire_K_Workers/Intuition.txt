Question Link : https://leetcode.com/problems/total-cost-to-hire-k-workers/description/

*   Problem:
        Hire k workers.
        At each step, you can choose from:
            - First "candidates" workers from left
            - First "candidates" workers from right
        Pick minimum cost worker.

*   Key observation:
        We only consider limited window
        from both ends at any time.

*   Core idea:
        Use two MIN HEAPS:
            1. Left heap
            2. Right heap

*   Why two heaps?
        Because candidates come from
        two independent regions.

*   Pointers:
        leftPtr → expands from start
        rightPtr → expands from end

*   Process for each hire:
        1. Fill left heap up to "candidates".
        2. Fill right heap up to "candidates".
        3. Compare smallest elements.
        4. Pick smaller cost.
        5. Remove it from its heap.
        6. Decrease k.

*   Why check leftPtr <= rightPtr?
        To avoid inserting same element twice.

*   Important invariant:
        Left heap contains up to "candidates"
        unused elements from left side.
        Right heap contains up to "candidates"
        unused elements from right side.

*   Greedy correctness:
        At each step,
        picking smallest available cost
        minimizes total cost.

*   Time Complexity:
        O(k log candidates)

*   Space Complexity:
        O(candidates)
